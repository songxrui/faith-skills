# faith-booknote — 20/80 深度读书笔记

> **synthesize** 先读完再产出。提取最有价值的 20% 观点 + 对应实践。人味在写法不在内容。
> 这是笔记，不是读后感。不是摘要，不是书评。

## 输入契约

```
书名 + 作者    [必填]
原文素材       [必填] 全文/章节扫描/PDF导出/划线
```

## 自适应切片逻辑

收到原文素材后，估算行数/字数：

- **≤500 行** → 一次读完，一次输出
- **500-1200 行** → 切为 2 部分，先读前半输出 → 再读后半追加
- **>1200 行** → 切为 3 部分，逐部分读-产出-追加

不需要询问用户。自动执行。

## 执行流程（5步，每步有完成标准）

### 步骤1: 初读 + 输出第一批观点

读取第1部分素材。
**完成标准**: 输出三层结构初版——骨架(核心论点+逻辑链) + 本阶段已发现观点(≥3个) + 概念索引初稿 + 临时标记(已读位置+已选观点编号)。

### 步骤2: 续读追加（如有第2部分）

读取第2部分素材。
**完成标准**: 新增观点(与步骤1不重复)已追加到第二层末尾，概念索引已更新，骨架已补充后半信息。输出"本阶段新增N个观点"。

### 步骤3: 续读追加（如有第3部分）

读取第3部分素材。
**完成标准**: 同上追加。如无新增观点则标注"本部分无新增观点"。

### 步骤4: 补完清理

检查并清理。
**完成标准**: (1)"(未完整阅读)"标注全部去除；(2)概念索引覆盖所有已提取观点；(3)观点总数≥5≤14。

### 步骤5: 最终输出

输出完整笔记。
**完成标准**: 三层结构完整 + 自检清单全部通过。

## 输出结构（固定三层）

### 第一层：全书骨架
一屏内说清核心论点、逻辑链、作者要解决的根本问题。用叙述保持密度。

### 第二层：20/80 核心观点 + 对应实践
5-14 个最有价值的观点。每观点包含：

```
观点[N]：一句核心观点
这个观点在说什么：1-2段书中有据的说明，保留原逻辑和关键论据
实践指导：落地到具体场景怎么用，具体到明天能试
```

规则：观点间按逻辑顺序排列，每观点标出处章节，引文用引用块。严禁对每观点套同一句式骨架。每段写法自然变化。

### 第三层：关键概念索引
书中核心概念，简短解释+出处。自动追加。

## 写作纪律

- 人味在句子节奏和选词上：避免"首先/值得注意的是/本质上/让我们"
- 实践指导必须具体到明天能做的程度
- 引文必须真实。不确定标「转述」
- 不编造作者没说过的话、不编造页码

## 门禁

- [ ] 全书是否全部读完？所有部分已追加？
- [ ] 观点 ≥5 个 ≤14 个？
- [ ] 每个观点的实践具体到明天能试？
- [ ] 有无 AI 味句式？有则改
- [ ] 引文出处真实标注？
- [ ] 是否写成了读后感（大段情绪无信息）？

## 失败模式

| 失败 | 兜底 |
|------|------|
| 找不到原文素材 | 问一句"书在哪里，给我原文" |
| 某部分无有价值观点 | 如实标注"本部分无新增观点" |
