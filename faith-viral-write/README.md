# Viral Writer ✍️

**基于 11 个内容洞见维度的自媒体创作 Skill，适用于各种智能体**

给它一个主题，它给你一篇完整的文章、5 个备选标题、以及每张配图的生成 prompt。

---

## 它能做什么

| 能力 | 说明 |
|------|------|
| 写完整文章 | 不是大纲，是可以直接发布的成品 |
| 多平台适配 | 微信公众号 / 小红书 / 抖音，三种完全不同的写法 |
| 标题生成 | 5 个备选标题，每个标注策略（好奇心缺口、数据冲击、痛点共鸣等） |
| 配图指导 | 封面图 + 正文配图的生成 prompt，可直接喂给 AI 图片生成工具 |
| Markdown 输出 | 结构化输出，方便后续编辑和排版 |

## 核心理念：11 个内容洞见维度

这个 Skill 的核心不是模板填空，而是让 AI **在创作前完成系统性思考**。每次创作，AI 会在内部走完 11 个维度的分析：

```
 1. 核心观点     — 一篇文章只需要一个锐利的观点
 2. 副观点       — 2-3 个角度让核心观点更立体
 3. 说服策略     — 数据驱动 / 故事驱动 / 权威背书 / 类比隐喻 / 社会认同
 4. 情绪触发点   — 共鸣、紧迫、好奇、优越、恐惧、归属
 5. 金句         — 能脱离上下文独立传播的记忆锚点
 6. 情感曲线     — 像电影一样设计内容的节奏起伏
 7. 情感层次     — 表层信息 → 中层价值观 → 深层身份认同
 8. 论证多样性   — 故事、数据、类比、反问交替使用
 9. 视角转化     — 第一/第二/第三人称的策略性切换
10. 语言风格     — 句长、口语化程度、修辞密度、节奏感
11. 互动钩子     — 让读者忍不住回复的设计（提问、填空、争议、投票）
```

这些维度不会展示给用户看——它们是创作引擎的"思考过程"，直接影响最终内容的深度和感染力。

## 快速开始

### 安装

将 `SKILL.md` 文件放入你的 Claude Code skills 目录：

```bash
# 克隆仓库
git clone https://github.com/nashsu/Viral_Writer_Skill.git

# 复制到 Claude Code skills 目录
cp -r Viral_Writer_Skill ~/.claude/skills/Viral_Writer_Skill
```

### 使用

安装后，在 Claude Code / OpenClaw / OpenCode 中直接用自然语言触发即可：

```
帮我写一篇公众号文章，主题是"为什么年轻人开始反消费主义"
```

```
写一篇小红书笔记，讲讲独居女生的周末早餐仪式感
```

```
帮我写个抖音文案，聊聊35岁转行到底值不值
```

你也可以指定更多细节：

```
写一篇公众号文章，主题是远程办公的利与弊，
目标受众是互联网从业者，风格偏理性深度
```

### 输出示例

每次创作会生成一个 Markdown 文件，结构如下：

```markdown
# 不是买不起，是想明白了：年轻人正在退出消费游戏

> 平台：微信公众号 | 字数：约2500字 | 调性：理性深度 + 情感共鸣

---

[完整正文]

---

## 备选标题

1. 不是买不起，是想明白了：年轻人正在退出消费游戏 — 策略：痛点共鸣
2. 月薪两万，全年消费不到三万：这届年轻人在想什么？ — 策略：数据冲击
3. 为什么你的购物车越来越空了？答案比你想的更深刻 — 策略：好奇心缺口
4. "不买立省百分百"：50万年轻人的反消费宣言 — 策略：社交货币
5. 你以为你在消费商品，其实商品在消费你的人生 — 策略：反常识

## 配图指导

### 封面图
- 推荐比例：2.35:1
- 视觉风格：极简主义，少即是多
- 生成 prompt：一个年轻人坐在空旷整洁的房间里，身边只有一杯茶和一本书，
  窗外是繁华的城市天际线，暖色调偏米白，扁平插画风格...

### 正文配图
#### 配图1（位置：第一节之后）
- 作用：情绪强化
- 生成 prompt：一个年轻人被堆积如山的购物袋包围淹没...
```

## 平台适配差异

这个 Skill 不是简单地把同一篇文章缩短或加长——不同平台是**完全不同的表达方式**：

| | 微信公众号 | 小红书 | 抖音 |
|---|---|---|---|
| 字数 | 1500-3000 | 300-800 | 200-500 |
| 语气 | 深度分析，有观点 | 闺蜜聊天，亲近感 | 面对面说话，短句快节奏 |
| 标题 | 15-30字，引发好奇 | 20字内，emoji辅助 | 前3秒抓人 |
| 配图 | 封面 + 2-4张正文图 | 封面决定点击率，3-6张图 | 分镜思路 + 画面描述 |
| 封面比例 | 2.35:1 | 3:4 | 9:16 |
| 互动 | 引导在看/转发 | 标签 + 评论互动 | 话题标签 + 评论引导 |

## 测试数据

我们对 3 个不同主题 × 3 个平台进行了 A/B 测试（with skill vs without skill）：

| 评估维度 | With Skill | Without Skill |
|----------|-----------|--------------|
| 综合通过率 | **100%** | 38% |
| 平台适配 | 全部符合平台规范 | 经常写成通用博客风格 |
| 备选标题 | 5个标题 + 策略标注 | 通常只有1个标题 |
| 配图指导 | 封面 + 正文配图 prompt | 无 |
| 互动钩子 | 100% 包含互动设计 | 33% |
| 金句质量 | 多处可独立传播的金句 | 偶尔有 |

## 工作原理

```
用户输入主题
     ↓
[第一步] 确认需求（平台、受众、调性）
     ↓
[第二步] 11 维度内容构思（内部思考，不展示）
     ↓
[第三步] 按平台规范创作正文
     ↓
[第四步] 生成 5 个备选标题 + 策略标注
     ↓
[第五步] 生成配图与封面指导（含 AI 生图 prompt）
     ↓
[第六步] 输出结构化 Markdown 文件
```

## 适合谁

- 自媒体从业者 / 内容运营
- 需要批量产出多平台内容的团队
- 想提升写作质量但缺乏方法论的创作者
- 任何需要"给我一个主题，还我一篇成品"的人

## 不适合什么

- 学术论文、技术文档等非自媒体场景
- 需要实时热点数据的时效性内容（AI 没有联网能力时）
- 替代人类创作者的独特风格和个人经历
