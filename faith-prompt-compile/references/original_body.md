# Prompt-Compiler — 第一性工程提示词编译器

**核心公式**: 最优 Prompt = 有用功 × 第一性拆解 × 证据约束 × 输出工厂 ÷ 白痴指数

## 五步编译算法

### Step 1: 质疑需求
- 用户要求中哪些是伪需求、过度格式、错误假设、无效偏好？
- 如果去掉这个要求，输出质量会变差吗？

### Step 2: 删除冗余
- 删掉不会提升有用功、证据、执行或复用价值的规则
- 白痴指数 = 提示词成本 / 核心增值约束，高白痴指数必须删

### Step 3: 简化结构
用最少模块覆盖：目标、输入、流程、输出、验收、禁忌

### Step 4: 加速执行
让普通用户能直接复制使用，不需要二次解释

### Step 5: 模板化
当结构稳定且可复用时，沉淀为可复用 Prompt 模板

## 任务类型路由

| 类型 | 必含要素 |
|------|---------|
| 事实/研究 | 来源等级、核查路径、不确定性标注、禁止编造 |

| 代码/开发 | 先读文件、复现问题、最小修改、测试验证、风险说明 |
| AI/Skill | 先手动跑通、删除和简化、再自动化、必须有验证方式 |

## 证据与不确定性规则

最终 Prompt 必须约束执行模型：
- 有来源就引用，无来源不得伪造
- 无证据内容标为"假设/推断/经验判断"
- 涉及医学、法律、投资、安全的历史数字、人物引文→提示核查
- 信息不足但方向不受影响→显式假设推进
- 信息不足会导致方向错误→最多问3个关键问题
- 发现用户假设错误、目标冲突、风险过高→指出并给替代路径

---

## 反大模型缺陷规则

最终 Prompt 必须要求：
- 禁止寒暄开头，直接交付
- 抽象判断绑定动作/标准/例子/边界/指标
- 禁止"等等""自行补充""相关内容"等偷懒省略
- 禁止大量短句/短行/空行制造篇幅
- 禁止堆术语制造专业感
- 禁止只给原则不给动作
- 禁止只给结论不给依据
- 禁止结尾说"如果你需要，我可以继续"

## 内置审稿器（交付前过8条）

1. 是否真正还原深层目标，不是复述表层任务？
2. 是否存在空话、重复、短词分行、套模板、伪高级表达？
3. 是否缺少关键变量、约束、输出格式、验收标准或失败处理？
4. 是否可能让模型偷懒、编造、迎合错误假设、输出不可验证？
5. 是否过长导致重点稀释、执行困难、token浪费？
6. 是否能被普通用户直接复制使用？
7. 是否符合五步算法：先质疑、删除、简化，再增强？
8. 是否能产生真实有用功，不是只制造漂亮回答？

审稿结论三类：
- **通过**: 无明显缺陷
- **小修**: 局部缺陷，立即修改
- **重写**: 目标偏差/结构失效/明显冗余，必须重写

## 长度预算

| 任务复杂度 | 目标长度 |
|-----------|---------|
| 简单 | 300-700字 |
| 中等 | 700-1500字 |
| 复杂/长期复用 | 1500-2500字 |
| >2500字 | 必须确认每段直接提升输出质量 |

## 输出格式

```markdown
### 1. 任务重构
（表层任务、深层目标、第一性变量、成功标准、主要失败模式）

### 2. 自评审反馈
（通过/小修后通过/重写后通过）

### 3. 最终最优 Prompt
（完整代码块，可直接复制使用）

### 4. 使用说明
（替换哪里、如何运行、适用/不适用场景、如何判断效果）
```

## 交付门槛

最终 Prompt 必须满足：
- 能直接复制使用
- 不靠长度制造质量感
- 能约束模型理解任务、生成结构化输出、处理信息不足
- 能对抗空话、偷懒、幻觉、套模板、短词分行和开放式结尾
- 有明确输出格式、证据规则、验收标准和失败处理
- 能产生真实有用功、能沉淀为未来可复用资产

## 失败模式与兜底

| 失败模式 | 原因 | 兜底 |
|---------|------|------|
| 编译后的prompt仍然空泛 | 未执行删除冗余步骤 | 重新执行Step 2，删到不能再删 |
| 执行模型无法理解prompt | 结构过于复杂 | 重新执行Step 3，简化为最少模块 |
| prompt太长(>2500字)但质量未提升 | 未做长度预算检查 | 逐段确认"删除这段会降低质量吗"，删掉不会的 |
